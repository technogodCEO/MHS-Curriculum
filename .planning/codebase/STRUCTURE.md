## Directory Purposes

**`react-app/src/`:**
- Purpose: All application source code
- Contains: Entry point, root component, reusable components, data files, baseline CSS

**`react-app/src/components/`:**
- Purpose: UI components that render sections of the curriculum viewer
- Contains: Five named-export `.jsx` components, each responsible for one visual region or panel
- Key files:
  - `CurriculumMap.jsx` — grade timeline for Math, English, Science, History, World Language
  - `CurriculumMapVariableTrackLengths.jsx` — sequential course timeline for Electives (variable-length tracks)
  - `ProgramOfStudies.jsx` — filterable course catalog (all subjects including electives)
  - `SubjectDropdownMenu.jsx` — absolute-positioned floating menu for switching subjects
  - `SubjectTitle.jsx` — stateless button rendering the current subject name with dropdown arrow

**`react-app/src/data/`:**
- Purpose: Static data modules; the entire curriculum content lives here
- Contains: One `.js` file per subject plus `subjects.js` as the master registry
- Key files:
  - `subjects.js` — the single source of truth for subject metadata; consumed by root component and `SubjectDropdownMenu`
  - `electives.js` — largest file (~119 KB); exports `electiveCategories`, `electiveTracks`, `electiveTrackColors`, and `electivesProgramOfStudies`
  - `math.js`, `science.js`, `english.js`, `history.js`, `language.js` — each exports `*Tracks`, `*TrackColors`, and `*ProgramOfStudies`

**`react-app/public/`:**
- Purpose: Static files copied verbatim into the CRA build output
- Contains: HTML shell, school logo image, PWA manifest, favicon
- Generated: No — hand-maintained

**`react-app/build/`:**
- Purpose: Production build artifact generated by `npm run build`
- Generated: Yes
- Committed: Yes (serves as the deployable artifact)

## Naming Conventions

**Files:**
- Components: PascalCase `.jsx` — e.g., `CurriculumMap.jsx`, `SubjectTitle.jsx`
- Data modules: camelCase `.js` — e.g., `math.js`, `subjects.js`
- CRA-generated utilities: camelCase `.js` — e.g., `reportWebVitals.js`, `setupTests.js`
- Root component file: snake_case `.jsx` — `math_curriculum_MHS.jsx` (legacy naming; does not match PascalCase convention of other components)

**Exports:**
- Components: Named exports from `components/` files — e.g., `export function CurriculumMap(...)`
- Root component: Default export — `export default function MathCurriculum()`
- Data: Named exports prefixed with subject name — e.g., `mathTracks`, `mathTrackColors`, `mathProgramOfStudies`

**Data export naming pattern:**
- `{subject}Tracks` — grade/track course map
- `{subject}TrackColors` — tab button display metadata
- `{subject}ProgramOfStudies` — full course catalog array
- Exception: Electives uses `electiveCategories` (extra export for category grouping) and `electiveTracks` (object-keyed, not array)

**CSS classes:**
- Kebab-case, defined as string literals injected via `<style>` JSX tags
- Examples: `.curriculum-header`, `.grade-card`, `.track-tab`, `.pos-filter-btn`
- Prefix `pos-` used for all Program of Studies-specific classes

## Where to Add New Code

**New Subject (e.g., "Physical Education"):**
1. Add a subject object to the `subjects` array in `react-app/src/data/subjects.js`
2. Create `react-app/src/data/phys_ed.js` exporting `physEdTracks`, `physEdTrackColors`, `physEdProgramOfStudies`
3. Add a `case 'phys_ed':` branch in the `subject()` switch inside `react-app/src/components/CurriculumMap.jsx`
4. Add a `case 'phys_ed':` branch in the `subject()` switch inside `react-app/src/components/ProgramOfStudies.jsx`
5. If tracks have variable lengths: add a case in `CurriculumMapVariableTrackLengths.jsx` and update the `activeSubject.id === "electives"` inline check in `react-app/src/math_curriculum_MHS.jsx`

**New UI Component:**
- Implementation: `react-app/src/components/NewComponent.jsx` as a named export
- Import: Add import line in `react-app/src/math_curriculum_MHS.jsx` (or the parent that uses it)

**New Shared CSS Classes:**
- Add to the `<style>` block inside `react-app/src/math_curriculum_MHS.jsx` if used by multiple components
- Add to the `<style>` block inside `react-app/src/components/ProgramOfStudies.jsx` if POS-only

**New Course in Existing Subject:**
- Add a course object to the appropriate `*ProgramOfStudies` array in the subject's data file (e.g., `react-app/src/data/math.js`)
- If it belongs on the Curriculum Map, add it to the relevant grade entry in `*Tracks`
