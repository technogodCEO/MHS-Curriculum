### Task 2: Remove `id` from elective track course objects (DEBT-01)

**Files:**
- Modify: `react-app/src/data/electives.js` (all course objects inside `electiveTracks.tracks`)

**Context:** Every course object nested under `electiveTracks.tracks[trackName].courses` has an `id` field (e.g. `id: "el-webdev"`). These IDs were used as React keys, but duplicate IDs across tracks cause React key collision warnings. The Program of Studies (`electivesProgramOfStudies`) is a completely separate export with its own `pos-*` IDs — it is NOT touched here.

**Step 1: Search for all `id:` fields in `electiveTracks`**

Open `react-app/src/data/electives.js`. The `electiveTracks` object starts at line 18 and ends around line 831. Every course object inside it has a line like:

```js
id: "el-ap-seminar", name: "AP Seminar", tier: "AP", ...
```

**Step 2: Delete every `id:` field from track course objects**

For each course in `electiveTracks.tracks`, remove the `id: "..."` property. Example before/after:

Before:
```js
{
  id: "el-webdev", name: "Webpage Design & Development", tier: "CP", weight: 0,
  gradeLevel: "Grades 9–12", credits: "2.5 (semester)",
```

After:
```js
{
  name: "Webpage Design & Development", tier: "CP", weight: 0,
  gradeLevel: "Grades 9–12", credits: "2.5 (semester)",
```

Do NOT touch the `electivesProgramOfStudies` export (starts at line 848) — those `pos-*` IDs are intentional and used by ProgramOfStudies.jsx.

**Step 3: Verify the edit with a grep — no `id:` should remain inside `electiveTracks`**

```bash
grep -n '"id":' react-app/src/data/electives.js
grep -n "id: \"el-" react-app/src/data/electives.js
```

Expected: no matches.
