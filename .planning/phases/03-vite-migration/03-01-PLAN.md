# Vite Migration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace Create React App (`react-scripts`) with Vite + Vitest in one atomic swap on the `vite-migration` branch.

**Architecture:** Remove `react-scripts` entirely. Install Vite as dev server/bundler and Vitest as the test runner. Vitest's Jest-compatible API means zero test file changes. All config lives in two new files (`vite.config.js`, `vitest.config.js`) plus a root-level `vercel.json`. Everything is committed in a single atomic commit.

**Tech Stack:** Vite 6, `@vitejs/plugin-react`, Vitest 3, jsdom, `@vitest/coverage-v8`

### Task 3: Create `vitest.config.js`

**Files:**
- Create: `react-app/vitest.config.js`

**Step 1: Create the file**

```js
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    setupFiles: './src/setupTests.js',
    globals: true,
  },
})
```

Notes:
- `environment: 'jsdom'` gives tests access to `document`, `window`, etc. â€” matches CRA's Jest setup
- `setupFiles` points to the existing `src/setupTests.js` (which imports `@testing-library/jest-dom`) â€” no changes to that file
- `globals: true` makes `describe`, `it`, `expect`, `beforeEach`, etc. available without imports â€” matches existing test style

### Task 5: Create `vercel.json`

**Files:**
- Create: `vercel.json` (at the **repo root**, not inside `react-app/`)

**Step 1: Create the file**

```json
{
  "buildCommand": "cd react-app && npm run build",
  "outputDirectory": "react-app/dist",
  "framework": null
}
```

Notes:
- `"framework": null` disables Vercel's CRA auto-detection
- `outputDirectory` points to `dist/` (Vite's default output) instead of CRA's `build/`
- `buildCommand` navigates into `react-app/` first since the project is not at the repo root
